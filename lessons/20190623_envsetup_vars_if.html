2019-06-23 - Dev environent setup, variables & if statements

  1. OS Stuff (linux windows)
    1. linux commands: ls, mkdir, rm, cd, pwd
    2. windows commands: dir, mkdir, rmdir, cd
    3. Directory structure and file names/paths
      1. relative vs fully qualified
      2. "." and ".." (dot and dot dot)
  2. Git - a version control system (vcs) - stores your code, tracks changes, etc.
    1. Repo - local vs remote
    2. windows git client (git bash)
      1. sorta like linux running on windows
    3. commands: clone, pull, push
    4. Workflow - pull, edit, commit, (repeat edit/commit), push
  3. Github - online service
    1. Is your remote repo
    2. repo creation and clone url
  4. Virtual Environments (aka venv)
    1. Why - b/c dependencies
    2. Where - in windows (not git bash)
    3. Creation: python -m venv <path_to_env>
    4. Activate: <path_to_env>/scripts/activate
    5. deactivate
  5. Text editor - Sublime Text
  6. Create & run a script
    1. create a file ending in .py
    2. write code
    3. run - at windows CLI (venv already activated): python <scriptname>
  7. Coding
    1. Write down what you want to build - in plan language - not code
    2. Think of how the user will interact w/your program
    3. Program does exactly what you tell it to do, nothing more
    4. "trick" - use comments to describe
  8. Python
    1. Comments
      1. Line type using #
      2. Docstrings (see strings section)
    2. Variables
      1. name that points to data
      2. Have a type, e.g. integer, string, float, etc.
      3. name can't have spaces
      4. name can't be the same as a keyword - e.g. can't call a variable "continue"
    3. Strings
      1. Quotes, single vs double
      2. Break across lines - just use quotes
      3. tripple quote strings (doc strings) - keep text formatting - e.g. line breaks
      4. Line termination - "new line" vs "carriage-return/line-feed" (unix vs windows)
    4. Operators
      1. Act on values/variables
      2. Math operators, +, -, *, etc.
      3. Assignment operator (=)
      4. Equality testing (==)
      5. "do the right thing" based on variable types, e.g. + will add numbers or concatenate strings
    5. If statement
      1. make a choice based on a condition/test, .e.g "if HP is greater than maxhp, then <do something>"
      2. "if", and optionally, "elif", "else"
    6. Indentation
      1. means - "inside" or "belongs to"
      2. Spaces NOT tabs (4 per level)
