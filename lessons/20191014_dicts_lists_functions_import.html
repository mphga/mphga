    4. Mutable vs Immutable

  3. Functions
    1. def <funcname>([params...]):
         <func code here>, e.g.
      1. def combat_loop(pcdata):
           #your code here
      2. function may take parameters (pcdata in the example above)
      3. you give the params any name you like - they become variables you can use in the function
      4. Variables you create in a function are only known to the function - if they overalp w/a global name the local name is used
    2. return
      1. This is a value you "give back to the caller"
      2. can be more than one - but only one should ever be called
      3. not all functions have to return a value (technically they all do, if you don't specify a return value - the function returns None)
    3. To call the function...
      myfunc(var1, var2)
  4. Modules/Packages
    1. import statement
      1. import <modulename>
        import yaml
      2. To use the imported code, you use prefix it with the module name, e.g.
        1. yaml.load()  # using the example import above
      3. you can also rename the import, like so
        1. import long_module_name as lmn
          lmn.function()
    2. Secific modules
      1. PyYAML
        1. load(yaml_thing), dump(data, yaml_thing)
          1. yaml_thing is either yaml in a string or - more commonly - yaml in a file
          2. data is some data you want to convert to yaml - commonly this is a dictionary
        2. Open file first if you're reading/writing (load/dump) from/to a file
    3. PyPI ("PIE-PEE-EYE")
      1. This is the website from which you can install packages - e.g. the PyYAML pagkage
      2. You, as human, can browse it to learn more about a given package - can also search
      3. as you progress you'll learn about more packages
    4. pip install
      1. this is how you install a package from PyPI
      2. best to follow this method - from inside your virtual environment...
        python -m pip install <modulename>
      3. Be sure to get the module name right, unfortunately the security of PyPI is weak, and others could upload malicious packages using the same, name as another, benign package
      4. pip freeze to see what packages are installed and the assocaited version
