2019-08-25 project directory structure and imports - for unittest

Python

  1. unittest
    1. project structure - to make unittest work
      1. <projdir>
            <projdir>
                __init__.py
                <driverscript>.py
                <libfile>.py
                <other_py_files>.py
                <other_non_py_files>.py
            <other_dirs>
            <test>
                __init__.py
                test_<somename>.py
                test_<othertestfiles>.py
      2. NB: the above structure may or may not be required - but it will enable unitest to work
      3. Calling unittest (give the above structure)

        python -m unittest

        ** issue the command above from inside the TOPLEVEL <projdir>
    2. imports
      1. in your test_<name>.py files you'll need to import your program files - in particular your lib file(s)
      2. import will look something like this:

        import <projname>.<libfilename>as <libfilename>

        e.g.

        import hp.lib as lib
  2. Standard Library
    1. os
      1. functions and such related to interacting with a computer operating system
      2. example we used was:

        os.path.join()

        which will join strings into a valid file path for whatever OS your program happens to be running on, which could be MacOS today, and Windows tomorrow
