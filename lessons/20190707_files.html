Python

  1. Files
    1. open(<fname>, <mode_string>)
      1. <fname> file name - as string
      2. <mode_string> is one of 'r' for read, 'w' for write, etc.
      3. REMEMBER: w will truncate the file, also will create if it doesn't exit
      4. REMEMBER 'r' requires the file to exist
      5. typcially use like: 'f = open('filename','r')
    2. file methods
      1. read(), readline(), write(), flush(), close()
      2. remember: you call on the file object, e.g. assume you opened a file and assigned it to the variable 'f', then, f.write('string to write'), etc.
      3. Assign the result of read functions to a variable to do something with the return values, e.g. data = f.readline()
      4. read() is the ENTIRE FILE - could be a problem if the file is huge
      5. write() writes exactly what you give it, if you intend what you write to be broken into lines, be sure the newline characters are where you expect them
      6. flush() ensures the data is written to the file - SORTA
        1. The OS can still hang onto it a while before actually writting to disk
      7. close() - does what it says
        1. if you want to flush, must do so before close
    3. Context manager - 'with'
      1. with open('filename', 'r') as myfile:
           myfile.readline()
      2. Why do you care: b/c it takes care of the flush/close clean up for you
      3. this is a general feature of Python and applies to more than just open() (degtails are not important - point is that it's not just for openening files)
